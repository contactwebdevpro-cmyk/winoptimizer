Déployer sur GitHub + Vercel

Mise en ligne en moins de 5 minutes

📁
Structure du repo & configuration Vercel
1
Créer le repo GitHub avec ces 4 fichiers
winoptimizer/
 ├── index.html     # Landing page
 ├── install.ps1    # Launcher (irm | iex)
 ├── WinOptimizer.ps1 # Script principal
 └── vercel.json    # Headers MIME ← important
2
Contenu du fichier vercel.json (obligatoire)
Sans ce fichier, Vercel ne sert pas correctement les .ps1 et irm échoue.
{
  "headers": [{
    "source": "/(.*).ps1",
    "headers": [{
      "key": "Content-Type",
      "value": "text/plain; charset=utf-8"
    }]
  }]
}
3
Importer sur Vercel
vercel.com → Add New Project → importer le repo GitHub. Dans les settings du projet, renommez le domaine en winoptimizer-powershell.
Framework Preset  : Other
Build Command    : (vide)
Output Directory : (vide)
